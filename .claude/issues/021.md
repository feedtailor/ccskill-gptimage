## 背景

#008(ccskill-gmail 型フルディスパッチャ)は #020 を踏まえてクローズしたが、その中で
**唯一、単独価値が残った機能が `doctor`** だった。当初は #010(コスト/モデレーション UX)へ
内包する案もあったが、診断はコスト UX とは関心が異なるため、独立 issue として切り出す。

`ccskill-gptimage doctor` は「なぜ画像生成が動かないのか」をユーザが自分で切り分けられるように、
環境と backend の状態を一覧診断する。#020 で実装済みの dispatcher は case 文への追加だけで
`doctor` を生やせる。

## 設計

`ccskill-gptimage doctor` を追加し、以下を ✓ / ✗ / ⚠ で表示する:

| 診断項目 | 判定方法 | 失敗時の案内 |
|---|---|---|
| Python venv | `$CCSKILL_GPTIMAGE_DIR/venv/bin/python` が存在し 3.10+ | `./install.sh` 再実行 |
| 依存パッケージ | venv の python で `import openai, dotenv` が通る | `./install.sh` 再実行 |
| PATH 配備 | `~/.local/bin/ccskill-gptimage` が dispatcher への symlink | `./install.sh` 再実行 |
| ユーザレベルスキル | `~/.claude/skills/ccskill-gptimage/SKILL.md` が解決可能 | `./install.sh` 再実行 |
| Codex backend | `codex` が PATH + `~/.codex/auth.json` に access_token | `brew install codex && codex login` |
| API backend | `OPENAI_API_KEY`(env or repo `.env`)が非空 | `cp .env.example .env` + キー設定 |
| backend 総合 | Codex / API のうち最低 1 つが利用可能 | 上記いずれかを設定 |
| Org Verification | **ベストエフォート**(後述) | Organization Settings で Verification |

### Org Verification 判定の方針

API キーがある場合のみ、軽量な検証を試みる。確実な判定は実 API 呼び出しを要するため:

- (案 A) `--deep` 指定時のみ、最小コストの呼び出し(または whoami 系エンドポイント)で 403 を検出
- (案 B) デフォルトは「キーの存在のみ確認し、403 の可能性を注記」

デフォルトは課金/レート消費を避けて案 B、`--deep` で案 A。実装時に OpenAI SDK の
非課金エンドポイントの有無を調査して確定する(推測で断定しない)。

### 終了コード

- backend 総合が ✗(Codex も API も無い)の場合は exit 1
- それ以外の個別 ⚠ は exit 0(警告のみ)
