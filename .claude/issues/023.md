### コードの実態(マージ済み)

PR #1 で `generate_image.py` に以下が追加され、**ファイル出力経路を優先しつつ、無い場合はセッションログから base64 を回収**するフォールバックになった:

- `CODEX_SESSIONS_DIR`(`~/.codex/sessions`)
- `_extract_codex_session_id()`: `codex exec` の stdout ヘッダ `session id: <uuid>` から session id を抽出
- `_find_codex_session_log()`: 対応する `rollout-*.jsonl` を特定(session id 優先、無ければ `since_mtime` 以降の最新ログ)
- `_recover_codex_image_from_session()`: `image_generation_call.result` の base64 を復号(`status == "completed"` 優先、`data:` URI も処理)
