## タスク概要

Phase 0 リリース後、初週内に Responses API 経路を追加する。`previous_response_id` を継承することで「画像を再アップロードせず、対話的に編集」する agentic 反復編集を実現 — これは nanobanana にない gpt-image-2 固有の差別化。

## 設計

### CLI

```bash
# 1ターン目: 通常生成(従来通り)
python generate_image.py "a gray tabby hugging an otter with orange scarf"
# → 出力: 20260423_153045.png + .json サイドカー(response_id を保存)

# 2ターン目: 続きから編集
python generate_image.py "Now make it photorealistic, golden hour lighting" \
  --continue ./generated_images/20260423_153045.png.json
# または
python generate_image.py "..." --continue <response_id>
```

### 内部実装

- `generate_image.py` に `_call_responses_api(client, prompt, previous_response_id=None, **)` を追加
- `--continue` が渡された場合のみ Responses API 経路に切替(なければ Image API)
- `OpenAI().responses.create(model="gpt-5", input=prompt, tools=[{"type": "image_generation"}], previous_response_id=...)`
- レスポンスから `image_generation_call` 出力を取り出し → `result.b64_json` を保存
- メタデータに `response_id` を必ず保存(次ターンの `--continue` で参照される)

### action パラメータ

- Responses API では `action`(`auto`/`generate`/`edit`)をオプション化(`--action`)。デフォルト `auto`

### model パラメータ

- Responses API は `model="gpt-5"` 等のメインライン経由が標準。`--model-host gpt-5` のような指定にする
