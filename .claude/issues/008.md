## タスク概要

ccskill-gmail のアーキテクチャに合わせて CLI ディスパッチャを導入。`ccskill-gptimage <subcommand>` 形式で setup/install/info/doctor/status/update/update-all/uninstall/register/release を提供する。

## 設計

```
ccskill-gptimage/
├── ccskill-gptimage          # ルート直下のディスパッチャ(bash)
├── commands/
│   ├── setup.sh              # venv 作成 + pip install
│   ├── install.sh            # 別プロジェクトへのインストール(symlink)
│   ├── uninstall.sh
│   ├── update.sh             # ディレクトリ単位の更新
│   ├── update-all.sh         # registry に基づく一括更新
│   ├── info.sh               # 環境/設定情報の表示
│   ├── status.sh             # 現状診断
│   ├── doctor.sh             # セットアップ問題のスキャン(API キー、Org Verify、依存)
│   ├── register.sh           # 既存インストールを registry に登録
│   └── release.sh
└── lib/
    ├── registry.sh           # .registry.json 操作
    ├── update_check.sh       # GitHub から新版チェック
    └── permissions.sh
```

## 既存スクリプトとの関係

- `generate_image.py` は据え置き(コア)。SKILL.md からの呼び出しは `$CCSKILL_GPTIMAGE_DIR/ccskill-gptimage generate ...` 経由に統一
- 直接 `python generate_image.py` も引き続き動く(後方互換)
